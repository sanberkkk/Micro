\section{Proje Özeti}
% ════════════════════════════════════════════════════════════════════════════

Bu proje kapsamında, MSP432 mikrodenetleyici tabanlı bir akıllı ekosistem sistemi
tasarlanmış ve gerçeklenmiştir. Sistem; termodinamik ve akustik sensörleri bir araya
getirerek sıradan bir yaşam alanını üç temel otonom işlev üzerinden yönetebilen bir
platforma dönüştürmektedir:

\begin{enumerate}[leftmargin=*, label=\arabic*.]
  \item \textbf{Soğuk Algılama ve Müzik Açma:} İçeceğin sıcaklığı hedef değere
        ulaştığında Bluetooth üzerinden otomatik müzik başlatılması.
  \item \textbf{Temassız Jest Kontrolü:} El kaydırma hareketiyle çalan şarkının
        ileri/geri değiştirilmesi.
  \item \textbf{Akustik Lamba Kontrolü:} El çırpma sesiyle ortam lambasının açılıp
        kapatılması ve parlaklık ayarı.
\end{enumerate}

% ════════════════════════════════════════════════════════════════════════════

% ════════════════════════════════════════════════════════════════════════════

Sistemin genel mimarisi Şekil~\ref{fig:mimari} üzerinde gösterilmiştir. MSP432 işlemcisi

birimleri yönetmektedir.

Sistem, sürekli bekleme döngüleri yerine \textbf{kesme (Interrupt) tabanlı} bir mimari
üzerine kurulmuştur. Her sensör olayı (sıcaklık eşiği, optik engelleme, ses darbesi)
ayrı bir donanımsal kesme vektörüne atanmıştır; bu sayede işlemci boşta kalırken
\texttt{\_\_low\_power\_mode\_0()} durumunda bekleyebilmekte ve gereksiz güç tüketimi
önlenmektedir.

\section{Özellik 1: Soğuk Algılama ve Müzik Açma}
% ════════════════════════════════════════════════════════════════════════════

\subsection{Çalışma Prensibi}

DS18B20 sıcaklık sensörü, 1-Wire protokolü üzerinden MSP432'ye periyodik olarak sıcaklık
verisi iletmektedir. Ölçüm döngüsü donanımsal Timer\_A kesmesi aracılığıyla
$\Delta t = 500\,\mathrm{ms}$'de bir tetiklenmektedir. Sensörden okunan ham değer
aşağıdaki dönüşüm formülüyle santigrat dereceye çevrilmektedir:

\begin{equation}
  T_{\mathrm{C}} = \frac{\mathrm{RawADC} \times V_{\mathrm{ref}}}{4096} \times k
  \label{eq:sicaklik}
\end{equation}

Dönüştürülen değer kullanıcı tarafından belirlenen $T_{\mathrm{eşik}}$ ile
kıyaslanmaktadır. Koşul sağlandığında sistem \texttt{IDLE} durumundan çıkarak
Bluetooth modülünü tetikler.

\subsection{Bluetooth Haberleşmesi (HC-05 / UART)}

MSP432'nin EUSCI\_A1 UART modülü, HC-05 modülüne $9600\,\mathrm{baud}$ hızında
bağlanmaktadır. Sıcaklık eşiği aşıldığında UART üzerinden \texttt{PLAY} komutu
gönderilmekte; bilgisayardaki Python betiği bu komutu alarak medya oynatıcıyı
başlatmaktadır.

\subsection{Kritik Kod Parçacığı}

\begin{lstlisting}[style=cstyle, caption={Sıcaklık eşiği kontrolü ve UART tetiklemesi}]
// Timer_A ISR - her 500 ms'de bir cagrilir
void TA0_0_IRQHandler(void) {
    TIMER_A0->CCTL[0] &= ~TIMER_A_CCTLN_CCIFG;

    float temp = DS18B20_ReadTemp();   // 1-Wire okuma

    if (temp <= TEMP_THRESHOLD && state == IDLE) {
        state = MUSIC_ON;
        UART_SendString("PLAY\r\n");   // HC-05 uzerinden komut gonder
        OLED_Print("Muzik Basladi");
    }
}
\end{lstlisting}

\subsection{Kalibrasyon ve Test Sonuçları}

Sensör, buz-su karışımında ($0\,^\circ\mathrm{C}$) ve oda sıcaklığında
($25\,^\circ\mathrm{C}$) kalibre edilmiştir. Ortalama ölçüm hatası
$\pm 0.3\,^\circ\mathrm{C}$ olarak belirlenmiştir.

% ════════════════════════════════════════════════════════════════════════════
\section{Özellik 2: Temassız Jest Kontrolü ile Müzik Değiştirme}
% ════════════════════════════════════════════════════════════════════════════

\subsection{Çalışma Prensibi}

İki adet LDR (ışığa bağlı direnç) sensörü birbirinden $d = 5\,\mathrm{cm}$ mesafede
yerleştirilmiştir. El sensörün üzerinden geçtiğinde her LDR sırayla gölgelenir;
gölgelenme zaman farkı ($\Delta t$) elin hareket \textit{yönünü} belirler:

\begin{itemize}[leftmargin=*]
  \item $\Delta t > 0$ (Sol $\to$ Sağ): Bir sonraki şarkıya geç $\to$ \texttt{NEXT}
  \item $\Delta t < 0$ (Sağ $\to$ Sol): Önceki şarkıya geç $\to$ \texttt{PREV}
\end{itemize}

\subsection{Çift-Kapı Doğrulama Algoritması}

Ortamdaki ani ışık değişimlerinin yanlış jest olarak algılanmaması için
\textbf{çift-kapı doğrulama} mekanizması uygulanmıştır:

\begin{enumerate}[leftmargin=*, label=\arabic*.]
  \item İlk LDR tetiklendiğinde donanımsal zamanlayıcı başlatılır.
  \item İkinci LDR belirlenen $t_{\max} = 300\,\mathrm{ms}$ penceresi içinde
        tetiklenirse jest geçerli sayılır.
  \item Pencere süresi aşılırsa olay görmezden gelinir ve zamanlayıcı sıfırlanır.
\end{enumerate}

\subsection{Kritik Kod Parçacığı}

\begin{lstlisting}[style=cstyle, caption={Çift-kapı jest doğrulama mantığı}]
// GPIO ISR - LDR1 tetiklendi
void PORT3_IRQHandler(void) {
    uint32_t status = P3->IFG;
    P3->IFG = 0;

    if (status & LDR1_PIN) {
        t_ldr1 = Timer32_GetValue(TIMER32_0_BASE);
        ldr1_triggered = true;
    }
    if (status & LDR2_PIN && ldr1_triggered) {
        uint32_t delta = t_ldr1 - Timer32_GetValue(TIMER32_0_BASE);
        if (delta < MAX_GESTURE_TICKS) {
            // Gecerli jest - yonu belirle ve komut gonder
            if (ldr1_first) UART_SendString("NEXT\r\n");
            else            UART_SendString("PREV\r\n");
        }
        ldr1_triggered = false;
    }
}
\end{lstlisting}

\subsection{Test Sonuçları}

50 tekrarlı test gerçekleştirilmiş; doğruluk oranı \%94 olarak ölçülmüştür.
Hatalı algılamaların tamamı $\Delta t > 280\,\mathrm{ms}$ sınır bölgesindeki
yavaş el hareketlerinden kaynaklanmıştır.

% ════════════════════════════════════════════════════════════════════════════
\section{Özellik 3: Akustik Lamba Kontrolü}
% ════════════════════════════════════════════════════════════════════════════

\subsection{Çalışma Prensibi}

KY-037 mikrofon modülü, ses sinyalinin genliği belirli bir eşiği aştığında dijital
çıkışını \texttt{HIGH} yapar. MSP432, bu pini harici kesme (External Interrupt) olarak
izlemektedir. El çırpma sesi algılandığında röle modülü \textit{toggle} edilerek
lamba açılır veya kapatılır.

\subsection{Gürültü Filtreleme}

Normal konuşma sesi ($\approx 60\,\mathrm{dB}$) ile el çırpma sesinin
($\approx 80\,\mathrm{dB}$) ayrıştırılması iki aşamalı bir yöntemle sağlanmıştır:

\begin{enumerate}[leftmargin=*, label=\arabic*.]
  \item \textbf{Donanımsal Eşikleme:} Mikrofon üzerindeki potansiyometre,
        yalnızca el çırpma düzeyindeki genliklerin tetikleme yapacağı şekilde
        kalibre edilmiştir.
  \item \textbf{Yazılımsal Bekleme Süresi (Debounce):} Kesme sonrası
        $t_{\mathrm{debounce}} = 200\,\mathrm{ms}$ boyunca yeni kesme kabul
        edilmemektedir; böylece eko ve çoklu darbe sorunları önlenmektedir.
\end{enumerate}

\subsection{Kritik Kod Parçacığı}

\begin{lstlisting}[style=cstyle, caption={Ses kesmesi ve röle toggle mantığı}]
volatile bool lamp_on = false;
volatile bool clap_ready = true;   // debounce bayragi

// Mikrofon GPIO ISR
void PORT5_IRQHandler(void) {
    P5->IFG = 0;

    if (!clap_ready) return;       // debounce aktifse yoksay

    // Lambay toggle et
    lamp_on = !lamp_on;
    if (lamp_on) P6->OUT |=  RELAY_PIN;
    else         P6->OUT &= ~RELAY_PIN;

    // 200 ms debounce - Timer32 ile
    clap_ready = false;
    Timer32_setCount(TIMER32_1_BASE, DEBOUNCE_TICKS);
    Timer32_startTimer(TIMER32_1_BASE, true);
}

// Debounce zamanlayici ISR
void T32_INT2_IRQHandler(void) {
    Timer32_clearInterruptFlag(TIMER32_1_BASE);
    clap_ready = true;
}
\end{lstlisting}

\subsection{Güvenlik Önlemleri}

Lamba bağlantısı izole bir röle kutusu içine alınmıştır. Geliştirme ve test
aşamasındaki ilk denemeler 12\,V DC düşük akımlı devreler üzerinde yürütülmüş;
yalnızca doğrulama sonrası 220\,V AC devresine geçilmiştir.

\section{Sonuç}
